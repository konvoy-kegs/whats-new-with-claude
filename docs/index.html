v2.1.166 — latest substantive release

Published 5 Jun 2026 at 19:01 UTC · GitHub release

Fallback models. New fallbackModel setting configures up to three fallback models tried in order when the primary model is overloaded or unavailable, and --fallback-model now applies to interactive sessions too. Claude Code also retries a turn once on the fallback model when the API rejects an unexpected non-retryable error. Useful resilience for CI and long-running agents during model brownouts.
Permission & cross-session hardening. Deny rules accept glob patterns in the tool-name position ("*" denies all tools), and messages relayed from other Claude sessions via SendMessage no longer carry user authority — receivers refuse relayed permission requests and auto mode blocks them.
  • feature fallbackModel setting — up to three ordered fallbacks when the primary model is overloaded or unavailable; --fallback-model now also applies to interactive sessions.
  • feature Glob pattern support in the deny-rule tool-name position ("*" denies all tools); allow rules reject non-MCP globs, and unknown tool names in deny rules warn at startup.
  • security Hardened cross-session messaging: messages relayed via SendMessage from other Claude sessions no longer carry user authority — relayed permission requests are refused and auto mode blocks them.
  • security Managed settings with an invalid entry no longer silently disable enforcement of their remaining valid policies.
  • security Managed-settings allowedMcpServers/deniedMcpServers predicates now match when they use ${VAR} references.
  • feature MAX_THINKING_TOKENS=0, --thinking disabled, and the per-model thinking toggle now disable thinking on models that think by default via the Claude API.
  • feature claude update announces the target version before downloading instead of going silent.
  • feature claude agents: typing a URL into the list filters to the session whose first prompt contained it.
  • fix Recurring "image could not be processed" error (and the extra token usage it caused) when an unprocessable image was sent in a session.
  • fix Remote sessions no longer become permanently stuck when a brief backend disruption occurs during worker registration at startup.
  • fix Windows: PowerShell command validation no longer hangs far past its time budget when a killed process's children hold its output pipes.
  • fix macOS: orphaned claude --bg-pty-host processes no longer spin at 100% CPU after the daemon dies.
  • fix Background agent sessions that entered a git worktree no longer crash-loop with "No conversation found" when reopened from claude agents.

v2.1.163 — managed version pinning

Published 4 Jun 2026 at 17:56 UTC · GitHub release

Pin the allowed Claude Code version org-wide. requiredMinimumVersion and requiredMaximumVersion managed settings let admins define an allowed version range — Claude Code refuses to start if its version falls outside it and directs the user to an approved version.
  • feature requiredMinimumVersion/requiredMaximumVersion managed settings to enforce an approved version range.
  • feature /plugin list command to list installed plugins, with --enabled/--disabled filters.
  • feature Stop and SubagentStop hooks can return hookSpecificOutput.additionalContext to give Claude feedback and keep the turn going.
  • security Fixed org-managed permission rules not applying for the entire session when the managed-settings fetch completed during startup on a fresh config directory.
  • security Fixed deny rules on home-directory paths (e.g. Read(~/Desktop/**)) not blocking Bash commands that reference the path via $HOME.
  • fix claude -p no longer hangs forever when a backgrounded command never exits, and no longer demands an Anthropic API key on Bedrock/Vertex/Foundry when CI=true.
  • fix Bash commands no longer fail under bazel and EDR-protected Go workflows ($TMPDIR regression in 2.1.154).

v2.1.162 — agents view & permission fixes

Published 3 Jun 2026 at 18:09 UTC · GitHub release

  • feature claude agents --json exposes waitingFor to surface what a waiting session is blocked on (e.g. a permission prompt).
  • security Windows permission rules now match backslash paths and case-variant paths; Read deny rules now hide files from Glob/Grep results.
  • security WebFetch permission rules now apply to built-in preapproved domains — explicit WebFetch(domain:...) rules take precedence over the preapproved-host auto-allow.
  • fix Silent startup hang when the config directory is read-only/unwritable — now starts with in-memory config and surfaces errors.
  • fix MCP per-server timeout values below 1000 ms no longer get floored to a 1-second watchdog that aborted every tool call.

v2.1.154 — major release headline

Published 28 May 2026 at 15:40 UTC · GitHub release

Opus 4.8 is here. Defaults to high effort. Use /effort xhigh for your hardest tasks. Fast mode on Opus 4.8 is now 2× standard rate for 2.5× the speed.
Dynamic workflows. Ask Claude to create a workflow and it orchestrates work across tens to hundreds of agents in the background. Run /workflows to view your runs. (Trigger keyword is now ultracode as of v2.1.160.)
  • feature The lean system prompt is now the default for all models except Haiku, Sonnet, and Opus 4.7 and earlier.
  • feature claude agents: type ! <command> to run a shell command as a background session. Also claude --bg --exec '<command>'.
  • security Improved the auto-mode classifier's detection of data exfiltration, particularly bulk transfers of repository contents.

Claude Desktop

macOS · Windows

What's new

Customize panel. A unified Customize section in Claude Desktop groups Skills, Plugins, and Connectors in one place — the same screen used to install our engineering plugin and GitHub connector.
