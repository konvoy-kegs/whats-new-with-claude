Claude Fable 5

model · access suspended

Access suspended by US government directive headline

Source: Statement on the US government directive to suspend access to Fable 5 and Mythos 5 · anthropic.com · 12 Jun 2026 · still in effect as of 16 Jun

Fable 5 and Mythos 5 remain offline for everyone. On 12 Jun the US government issued an export-control directive, citing national security, to suspend all access to Fable 5 and Mythos 5 by any foreign national — inside or outside the US. To comply, Anthropic disabled both models for all customers across every surface (Claude apps, API, Amazon Bedrock, Vertex AI, Microsoft Foundry, GitHub Copilot). As of this morning there is no restoration notice. Access to all other Anthropic models — Opus 4.8, Sonnet, Haiku — is unaffected.
What this means for us: any workflow pinned to claude-fable-5 (CLI sessions, Bedrock calls, CI agents) will still fail. Point those back at claude-opus-4-8 until access is restored. Claude Code lists Fable 5 in the /model picker but selecting it errors while the suspension is in effect.
  • status Anthropic says it is complying with the legal directive but disagrees with it — it believes the order stems from a narrow, non-universal “jailbreak” (asking the model to read a codebase and fix flaws) whose capability is widely available from other public models, and is “working to restore access as soon as possible.”
  • background Fable 5 launched 9 Jun as a Mythos-class model made safe for general use, with safeguards that auto-route cybersecurity, biology/chemistry, and distillation queries to Opus 4.8. Mythos 5 is the same model with those safeguards lifted, restricted to Project Glasswing partners.
  • pricing (while it was live) $10 / M input and $50 / M output — about double Opus 4.8 — with a 30-day data-retention requirement on all Mythos-class traffic. These terms are moot until access returns.
  • note The earlier “free on plan through 22 Jun” promotion is superseded by the suspension. We'll track restoration here as it develops.

v2.1.178 — parameter-scoped permission rules & tighter auto mode newest

Published 15 Jun 2026 at 19:03 UTC · GitHub release

Permission rules can now match a tool's input parameters. New Tool(param:value) syntax (with a * wildcard) lets a rule target specific arguments — e.g. Agent(model:opus) to block Opus subagents. A finer-grained governance lever than allow/deny by tool name alone.
Auto mode now reviews subagents before they launch. Subagent spawns are evaluated by the classifier before launch, closing a gap where a subagent could request a blocked action without review. Worth noting if you run auto mode in CI or shared environments.
  • feature Tool(param:value) permission-rule syntax with * wildcard to match tool input parameters (e.g. Agent(model:opus)).
  • security Auto mode evaluates subagent spawns with the classifier before launch — a subagent can no longer request a blocked action without review.
  • feature Skills in nested .claude/skills directories now load when working on files there; on a name clash the nested skill appears as <dir>:<name> so both stay available. The agent/workflow/output-style closest to the working directory now wins on collision.
  • security Fixed Claude in Chrome silently failing to connect when the OAuth token belongs to a different account than the Claude Code login.
  • security Fixed MCP server-level specs (mcp__server, mcp__server__*, mcp__*) in subagent disallowedTools being silently ignored.
  • fix Compaction now honors --fallback-model: it falls back to the configured model chain on overload or model-availability errors.
  • fix Fixed model requests still failing with auth errors after credentials were refreshed outside the session (stale cached request config), and an out-of-memory crash from inheriting a stale websocket/OAuth file descriptor.
  • fix Subagents: transcripts now show tool results and live progress, messages sent while a subagent finishes its turn are no longer dropped, and backgrounding a running subagent (ctrl+b) no longer restarts it.

v2.1.176 — model-allowlist enforcement, Bedrock creds & hook path fixes

Published 12 Jun 2026 at 19:45 UTC · GitHub release

Model allowlists are harder to bypass. Alias model picks can no longer be redirected to a blocked model via ANTHROPIC_DEFAULT_*_MODEL env vars, and /fast now refuses to toggle when it would switch to a model outside the availableModels allowlist. Relevant if we govern which models the team is allowed to run.
Bedrock credential caching. Credentials from awsCredentialExport are now cached until their Expiration instead of a fixed 1 hour — fewer re-auths during long Bedrock sessions.
  • security availableModels enforcement: alias picks can't be redirected to a blocked model via env vars; /fast refuses to switch to a disallowed model.
  • feature Bedrock credentials from awsCredentialExport cached until their Expiration rather than a fixed hour.
  • security Hook if conditions for Read/Edit/Write tool paths now match documented patterns — e.g. Edit(src/**), Read(~/.ssh/**), Read(.env).
  • fix Linux sandbox failing to start when .claude/settings.json is a symlink with an absolute target.
  • feature Session titles generated in the language of your conversation (pin via the language setting); new footerLinksRegexes setting for regex-matched footer link badges.
  • fix Auto mode failing on Fable 5 for orgs without Opus 4.8 — the classifier now falls back to the best available Opus model.
  • fix Several Remote Control fixes: connecting from web/mobile silently switching the session model, bare numeric disconnect codes, and sessions not disconnecting on sign-in to a different account.
  • fix /copy and mouse-selection copy now reach the system clipboard inside tmux over SSH.

v2.1.175 — enforce the model allowlist on Default

Published 12 Jun 2026 at 02:10 UTC · GitHub release

New enforceAvailableModels managed setting. When enabled, the availableModels allowlist also constrains the Default model — a Default that would resolve to a disallowed model falls back to the first allowed one, and user or project settings can no longer widen a managed list. A clean governance lever for locking the team to approved models.

v2.1.174 — /model picker & Bedrock GovCloud fixes

Published 11 Jun 2026 at 22:27 UTC · GitHub release

Per-skill/agent usage attribution in VSCode. The Account & usage dialog (/usage) now breaks down cache misses, long context, subagents, and per-skill/agent/plugin/MCP usage over the last 24h or 7d.
  • fix /model picker now shows the model family Default resolves to — Opus on Max/Team Premium/Enterprise, Sonnet on Pro/Team, Opus on PAYG API.
  • security Bedrock GovCloud regions (us-gov-*) were deriving the wrong inference-profile prefix (global instead of us-gov), causing 400 errors — fixed.
  • fix Background sessions no longer inherit another session's ANTHROPIC_* provider env (gateway URL, headers, model aliases).
  • feature New wheelScrollAccelerationEnabled setting to disable mouse-wheel scroll acceleration in fullscreen mode.

v2.1.172 — nested sub-agents & Bedrock region from ~/.aws

Published 10 Jun 2026 at 17:55 UTC · GitHub release

Sub-agents all the way down. Sub-agents can now spawn their own sub-agents, up to 5 levels deep — deeper delegation trees for orchestrating large multi-agent tasks.
Bedrock region from ~/.aws. Amazon Bedrock now reads the AWS region from ~/.aws config files when AWS_REGION isn't set, matching AWS SDK precedence — and /status shows where the region came from.
  • feature Sub-agents can spawn their own sub-agents (up to 5 levels deep); search bar when browsing a marketplace's plugins in /plugin.
  • security Fixed availableModels restrictions not applying to subagent model overrides, the dispatch model picker, and the advisor model; version-specific IDs no longer hide /model rows.
  • security Fixed WebFetch(domain:*.example.com) wildcard rules never matching subdomains, and file rules with mid-pattern wildcards (e.g. Read(secrets-*/config.json)) being rejected at startup.
  • fix Sessions using 1M context without usage credits no longer get permanently stuck — they auto-compact back under the standard limit.
  • fix Performance: less redundant message processing in long conversations, lower idle CPU, batched Claude in Chrome tool loading.

Claude Desktop

macOS · Windows

What's new

Claude Cowork

Generally available

What's new

Sources: Getting started with Cowork, unified directory docs, Financial Services plugins, Use plugins in Claude. No new Cowork release in the last 7 days; items below remain current.

Projects. Organize related tasks into persistent, self-contained workspaces — each with its own files, links, instructions, and memory — making Cowork more powerful for recurring or long-running work.
Scheduled tasks. Delegate work to Cowork on a recurring or on-demand basis — type /schedule in chat to create one. This morning's digest is exactly this in action.
  • feature Plugins bundle skills, connectors, and sub-agents into one installable package. Knowledge Work marketplace is default; Financial Services and Legal are add-ons, and you can install custom marketplaces from a GitHub repo.
  • feature Ten ready-to-run Financial Services agent templates (pitchbooks, KYC screening, month-end close) ship as plugins for Cowork and Claude Code.
  • feature Usage analytics for Cowork via the Analytics API, plus OpenTelemetry support for monitoring.
  • feature Skills, Connectors, and Plugins all browseable from a unified directory in Settings.
  • feature Computer use — Claude can click, type, open apps, and run dev tools in Cowork (research preview on Pro and Max).
  • feature Admins can group users (manually or via SCIM) and assign custom roles defining which Claude capabilities each group can use.
  • feature Extend Cowork with third-party MCP platforms (Slack, Sentry, AWS, Atlassian, etc.).
