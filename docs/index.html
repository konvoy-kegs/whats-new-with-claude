Claude Fable 5

model · access suspended

Access suspended by US government directive headline

Source: Statement on the US government directive to suspend access to Fable 5 and Mythos 5 · anthropic.com · 12 Jun 2026 · still in effect as of 29 Jun

Fable 5 and Mythos 5 remain offline for everyone. On 12 Jun the US government issued an export-control directive, citing national security, to suspend all access to Fable 5 and Mythos 5 by any foreign national — inside or outside the US. To comply, Anthropic disabled both models for all customers across every surface (Claude apps, API, Amazon Bedrock, Vertex AI, Microsoft Foundry, GitHub Copilot). As of this morning (29 Jun) there is still no restoration notice and no confirmed restoration date — that is now 17 days of suspension — though Anthropic has said it is “working to restore access as soon as possible.” Access to all other Anthropic models — Opus 4.8, Sonnet, Haiku — is unaffected.
What this means for us: any workflow pinned to claude-fable-5 (CLI sessions, Bedrock calls, CI agents) will still fail. Point those back at claude-opus-4-8 until access is restored. Claude Code lists Fable 5 in the /model picker but selecting it errors while the suspension is in effect.
  • status Anthropic says it is complying with the legal directive but disagrees with it — it reviewed the government’s demonstration and characterises it as a small number of previously known, minor vulnerabilities rather than a novel capability, and is “working to restore access as soon as possible.” It also intends to restore Fable 5 as a standard part of subscription plans once access returns.
  • background Fable 5 launched 9 Jun as a Mythos-class model made safe for general use, with safeguards that auto-route cybersecurity, biology/chemistry, and distillation queries to Opus 4.8. Mythos 5 is the same model with those safeguards lifted, restricted to Project Glasswing partners.
  • pricing (while it was live) $10 / M input and $50 / M output — about double Opus 4.8 — with a 30-day data-retention requirement on all Mythos-class traffic. These terms are moot until access returns.
  • note The earlier “free on plan through 22 Jun” promotion is superseded by the suspension. We'll track restoration here as it develops.

v2.1.195 — hook matchers now exact-match & project settings can’t auto-load external plugins newest

Published 26 Jun 2026 at 18:16 UTC · GitHub release · a hook-matching behaviour change, a plugin install-consent fix, and a background-agent reliability sweep

Hook matchers with hyphenated identifiers now exact-match — worth auditing our hooks. Previously a matcher like code-reviewer or mcp__brave-search could accidentally substring-match other identifiers; v2.1.195 makes them exact-match. If you want to match every tool from a hyphenated MCP server, use the explicit pattern mcp__brave-search__.*. Anyone running hooks keyed to hyphenated agent or MCP-server names should check they still fire as intended after upgrading.
A project’s .claude/settings.json can no longer silently enable an external plugin. Fixed a path where external plugins enabled only by a project’s .claude/settings.json didn’t require explicit install consent on every loader path — so cloning a repo that ships a crafted settings file can’t auto-load an external plugin without you approving it. Directly relevant to anyone cloning or running shared repos in Claude Code.
  • consent External plugins enabled only by a project .claude/settings.json now require explicit install consent on every loader path; /plugin Enable/Disable now works when a plugin’s plugin.json name differs from its marketplace entry name.
  • hooks Hook matchers with hyphenated identifiers (e.g. code-reviewer, mcp__brave-search) now exact-match instead of substring-matching — use mcp__brave-search__.* to match all tools from a hyphenated MCP server.
  • background agents Background jobs no longer disappear from claude agents or lose data when written by a newer Claude Code version; reopening a crashed background task no longer shows a blank screen for up to 5s; background-agent daemons no longer run unreachable when the control socket fails to start (which had blocked restarts).
  • feature New CLAUDE_CODE_DISABLE_MOUSE_CLICKS disables mouse click/drag/hover in fullscreen mode while keeping wheel scroll; Remote session startup now shows a provisioning checklist while the container starts.
  • voice Voice dictation on macOS no longer captures silence after the default input device changes; auto-submit now fires for languages written without spaces (Japanese, Chinese, Thai); Linux voice mode now distinguishes “no microphone” from “SoX not installed.”

v2.1.193 — route all shell through the auto-mode classifier & a new OTel response-text log event

Published 25 Jun 2026 at 19:05 UTC · GitHub release · auto-mode hardening, an observability change with an upgrade gotcha, MCP auth resilience, and a background-agent fix sweep

New autoMode.classifyAllShell routes every Bash/PowerShell command through the auto-mode classifier. Previously auto mode only classified commands matching arbitrary-code-execution patterns; set autoMode.classifyAllShell and all shell commands are run past the classifier first. Worth turning on anywhere we let Claude Code run unattended against infra, CI, or shared repos. Auto-mode denial reasons now also show up in the transcript, the denial toast, and /permissions recent denials, so it's clearer why a command was blocked.
Heads-up for our OpenTelemetry setup: a new log event can start capturing Claude's response text on upgrade. v2.1.193 adds a claude_code.assistant_response OTel log event containing the model's response text. It's redacted unless OTEL_LOG_ASSISTANT_RESPONSES=1 — but when that var is unset it follows OTEL_LOG_USER_PROMPTS. So any deployment that already logs prompt content will begin receiving response content automatically on upgrade. If we want to stay prompts-only, set OTEL_LOG_ASSISTANT_RESPONSES=0 before rolling this out.
  • auto mode New autoMode.classifyAllShell setting routes all Bash/PowerShell commands through the auto-mode classifier instead of only arbitrary-code-execution patterns; denial reasons added to the transcript, denial toast, and /permissions recent denials.
  • telemetry New claude_code.assistant_response OpenTelemetry log event carrying the model's response text — redacted unless OTEL_LOG_ASSISTANT_RESPONSES=1, and when unset it follows OTEL_LOG_USER_PROMPTS (set OTEL_LOG_ASSISTANT_RESPONSES=0 to keep prompts-only).
  • feature Live file-path autocomplete in bash mode (!); a startup notice when MCP servers need authentication, pointing at /mcp; automatic memory-pressure reaping for idle background shell commands (disable with CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP=1).
  • fix Background-agent sweep: backgrounding (←←) no longer spuriously cancels when all tasks carry over to the new session; pinned background agents are no longer re-prompted to “Continue from where you left off” after every auto-update; backgrounding the main turn no longer spawns a phantom “general-purpose (resumed)” subagent.
  • reliability MCP headersHelper auth now re-runs and reconnects automatically when a tool call returns 401/403; plugin auto-rename now follows marketplace renames maps; /model and other client-data-gated UI no longer show stale/empty state right after /login.

v2.1.191 — /rewind past a /clear, permanent agent stop & a big perf pass

Published 24 Jun 2026 at 18:54 UTC · GitHub release · a new recovery workflow, agent-lifecycle fix, streaming perf, and a large fix sweep

New /rewind can resume a conversation from before /clear was run. Cleared a session you didn't mean to? /rewind now restores the conversation state from before the /clear — a genuine undo for the one command that used to be irreversible.
Stopping a background agent is now permanent. Background agents no longer resurrect after being stopped — stopping one from the tasks panel sticks. Paired with a ~37% cut in CPU usage during streaming responses (text updates coalesced to 100ms) and reduced long-session memory growth, this is a solid reliability/perf release for anyone running long or concurrent agent sessions.
  • feature /rewind support for resuming a conversation from before /clear was run.
  • feature Sandbox network permission dialog now remembers hosts you allow with “Yes” for the rest of the session instead of re-prompting on every connection.
  • governance Managed settings: forceRemoteSettingsRefresh now takes effect when set via MDM or file policy, and the fetch sends Cache-Control: no-cache so proxies can't serve stale managed config — relevant for our MDM-pushed Claude Code policy.
  • reliability MCP capability discovery (tools/list, prompts/list, resources/list) now retries transient network errors with backoff; MCP OAuth discovery/token requests retry once after transient errors, and headless environments skip the browser popup and go straight to the paste-the-URL prompt; HTTP 404 errors now show the URL and point at your MCP config.
  • fix Background agents no longer resurrect after being stopped; hooks with comma-separated matchers (e.g. "Bash,PowerShell") now fire; /permissions approving a recently-denied tool now persists on close; /voice now explains an org-policy restriction instead of a generic “not available” message; plus terminal/scroll/Windows Terminal fixes.

v2.1.187 — block sandboxed commands from reading secrets & org-enforced model restrictions

Published 23 Jun 2026 at 17:55 UTC · GitHub release · a new sandbox-credentials lock, org model governance, and a large fix sweep

New sandbox.credentials setting stops sandboxed commands reading your secrets. Set it to block sandboxed commands from reading credential files and secret environment variables — so a command running in the sandbox can’t quietly exfiltrate ~/.aws/credentials, .env files, or tokens from the environment. Worth turning on anywhere we run Claude Code against repos that touch our AWS or CI secrets.
Org-configured model restrictions now reach every entry point. The model picker, --model, /model, and the ANTHROPIC_MODEL env var all now honour org-configured model restrictions, showing a “restricted by your organization’s settings” message when a blocked model is selected. Combined with the earlier enforceAvailableModels managed setting, this closes the last few ways a user or env var could reach a model the org hasn’t approved.
  • security New sandbox.credentials setting blocks sandboxed commands from reading credential files and secret environment variables.
  • feature Org-configured model restrictions now apply to the model picker, --model, /model, and ANTHROPIC_MODEL, with a “restricted by your organization’s settings” message; mouse-click support added to select menus (permission prompts, /model, /config) in fullscreen mode.
  • change /install-github-app — GitHub Actions workflow setup is now optional, so you can install just the GitHub App and skip the workflow/secret steps.
  • fix Remote MCP tool calls that hang with no response for 5 minutes now abort with an error instead of blocking indefinitely (override with CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT); --json-schema / workflow agent({schema}) structured output no longer re-calls StructuredOutput indefinitely; --resume no longer fails with “No conversation found” when the original -p run produced no model turns.
  • fix Background jobs in the agents view no longer get stuck in “working”; channel connections no longer drop after navigating to the agents view or after /bg, /tui, or /update; subagent depth tracking and leaked agent worktree registrations fixed; [VSCode] extension no longer becomes unresponsive when resuming a large session.

v2.1.186 — authenticate MCP servers from the CLI & permission-rule fixes

Published 22 Jun 2026 at 18:03 UTC · GitHub release · new MCP auth commands, permission/sandbox fixes, a large fix sweep

Log in to MCP servers without opening the /mcp menu. New claude mcp login <name> and claude mcp logout <name> commands authenticate MCP servers straight from the CLI, with --no-browser stdin-redirect support so you can complete an OAuth flow over SSH. Handy for headless boxes and CI runners where the interactive menu isn't an option.
Three permission / sandbox enforcement gaps closed. Agent(type) deny rules and Agent(x,y) allowed-types restrictions are now enforced for named subagent spawns; --tools no longer lets feature-gated tools slip through before flags load on a cold first launch; and Chrome tab-group isolation now applies even when the in-product permissions gate is off for concurrent CLI sessions. Worth knowing if you rely on Agent() rules to fence off subagents.
  • security Agent(type) deny rules and Agent(x,y) allowed-types restrictions now enforced for named subagent spawns; --tools no longer lets feature-gated tools slip through on a cold first launch; Chrome tab-group isolation now applies when the in-product permissions gate is off.
  • feature claude mcp login <name> / claude mcp logout <name> to authenticate MCP servers from the CLI, with --no-browser for completing over SSH; new “Skills” section on the /plugin Installed tab; status filtering (press f) in the /workflows agent detail view.
  • feature ! bash commands now trigger Claude to respond to the output automatically — set "respondToBashCommands": false to keep the old context-only behaviour; new “Claude Platform on AWS — refresh credentials” option in /login when awsAuthRefresh is configured; new teammateMode: "iterm2".
  • change Background subagents now surface permission prompts in the main session instead of auto-denying — the dialog shows which agent is asking, and Esc denies just that tool.
  • change CLAUDE_CODE_MAX_RETRIES now caps at 15 — for unattended sessions use CLAUDE_CODE_RETRY_WATCHDOG instead; /review <pr> now uses the same review engine as /code-review medium.

v2.1.183 — auto mode won't run destructive git / IaC commands you didn't ask for still relevant

Published 19 Jun 2026 at 00:02 UTC · GitHub release · safety hardening + new settings + a fix sweep

Auto mode now blocks destructive commands you didn't request. git reset --hard, git checkout -- ., git clean -fd and git stash drop are blocked unless you asked to discard local work; git commit --amend is blocked when the commit wasn't made by the agent this session; and terraform destroy / pulumi destroy / cdk destroy are blocked unless you named the specific stack. Directly relevant to anyone pointing Claude Code at our Terraform/CDK or shared repos in auto mode.
Automated triggers can no longer approve actions. Scheduled-task and webhook trigger deliveries are no longer treated as keyboard input — they now classify as task notifications and can no longer approve a pending action or set the session title in auto mode, closing a path where an automated trigger (like this very digest) could have rubber-stamped a prompt. MCP servers requiring authentication also no longer expose auth-stub tools to the model in headless/SDK mode.
  • security Auto mode blocks destructive git commands you didn't ask for, and terraform/pulumi/cdk destroy unless you named the stack; git commit --amend blocked for commits not made by the agent this session.
  • security Scheduled-task / webhook trigger deliveries reclassified as task notifications — can't approve actions or set the session title in auto mode; MCP servers requiring auth no longer expose auth-stub tools to the model in headless/SDK mode.
  • feature Warning when the requested model is deprecated or auto-updated to a newer one (on stderr in print mode -p, now also covering models set in agent frontmatter); new attribution.sessionUrl setting to omit the claude.ai session link from commits and PRs.
  • feature /config --help lists all shorthand keys; /config toggle now changes a setting with Enter or Space, and Esc saves and closes instead of reverting.
  • fix Fixed WebSearch returning empty results in subagents, and thinking.disabled.display 400 errors on subagent spawns and session-title generation.
