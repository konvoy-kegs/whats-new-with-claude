v2.1.161 — today's headline release

Published 2 Jun 2026 at 17:38 UTC · GitHub release

Credential leak fixed in claude mcp. claude mcp list/get/add no longer print secrets to the terminal: ${VAR} references are no longer expanded, and credential headers and URL secrets are now redacted. Worth a glance if anyone has piped claude mcp output into logs, CI, or screenshots.
Slice usage metrics by team or repo. OTEL_RESOURCE_ATTRIBUTES values are now attached as labels on metric datapoints, so you can break Claude Code usage down by custom dimensions like team or repository in your existing observability stack.
  • security claude mcp list/get/add no longer print secrets — ${VAR} references unexpanded; credential headers and URL secrets redacted.
  • feature OTEL_RESOURCE_ATTRIBUTES values now exported as labels on metric datapoints — slice usage by team or repo.
  • feature /mcp now collapses claude.ai connectors you've never signed in to behind a "Show unused connectors" row.
  • feature claude agents rows show done/total when work is fanned out; peek surfaces the longest-running item.
  • fix Parallel tool calls: a failed Bash command no longer cancels the other calls in the same batch — each tool returns its own result.
  • fix forceLoginOrgUUID/forceLoginMethod managed-settings policies no longer block third-party provider sessions (Bedrock, Vertex, Foundry, Mantle) alongside the org pin (regression in 2.1.146).
  • fix Workflow agents spawned with isolation: "worktree" in background sessions can now edit files inside their own worktree.
  • fix Background subagent output no longer corrupts claude -p stdout under --output-format text or json.
  • fix OpenTelemetry log events (user_prompt, api_request, tool_result, tool_decision) are no longer dropped when emitted before telemetry init completes.
  • fix /usage-credits no longer starts a re-login for Team/Enterprise admins — it points to the org's usage settings page.
  • fix /autofix-pr no longer reports "cannot run on the default branch" when the session is inside a git worktree or another repo.
  • fix Windows hooks that invoke bash explicitly (e.g. /usr/bin/bash script.sh) no longer fail with "command not found".
  • fix Linux fullscreen clipboard now uses wl-copy/xclip/xsel when available, copying to both the clipboard and PRIMARY selection.
  • fix Improved rendering performance for large file writes and stabilized the layout engine's JIT profile.

v2.1.160 — previous user-facing release

Published 1 Jun 2026 at 20:03 UTC · GitHub release

New write-path security prompts. Claude now prompts before writing to shell startup files (.zshenv, .zlogin, .bash_login) and ~/.config/git/ — closing a quiet path to unintended command execution. acceptEdits mode now also prompts before touching build-tool config files that grant code execution (.npmrc, .yarnrc*, bunfig.toml, .bazelrc, .pre-commit-config.yaml, .devcontainer/).
Workflow trigger renamed to ultracode. Typing the word "workflow" no longer triggers a dynamic-workflow run — the trigger keyword is now ultracode (highlighted in violet in the prompt input). Asking for a workflow in your own words still works.
  • security Prompt before writing to shell startup files and ~/.config/git/; acceptEdits prompts before code-executing build-tool config files.
  • feature Edit no longer requires a separate Read after viewing a file with grep: single-file grep/egrep/fgrep now satisfy the read-before-edit check.
  • fix Restoring a completed claude agents session no longer drops chat history and re-runs the original prompt.
  • fix Background sessions re-attached after overnight retire no longer lose conversation and re-run the original prompt.
  • fix Copy-on-select now writes to the Windows clipboard on WSL (PowerShell interop replaces OSC 52).
  • fix Many claude agents stability fixes on Windows under heavy CPU load.

Removed: CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE is now a no-op. The JetBrains plugin install suggestion at startup has also been removed.

v2.1.158 — auto mode on managed clouds

Published 30 May 2026 at 01:12 UTC · GitHub release

Auto mode lands on Bedrock, Vertex, and Foundry. Now available for Opus 4.7 and Opus 4.8 on all three managed-cloud backends. Opt in by setting CLAUDE_CODE_ENABLE_AUTO_MODE=1. Unblocks enterprise teams pinned to non-Anthropic-hosted inference for compliance reasons.
  • feature Auto mode now available on Bedrock, Vertex, and Foundry for Opus 4.7 and Opus 4.8.

v2.1.154 — major release headline

Published 28 May 2026 at 15:40 UTC · GitHub release

Opus 4.8 is here. Defaults to high effort. Use /effort xhigh for your hardest tasks. Fast mode on Opus 4.8 is now 2× standard rate for 2.5× the speed.
Dynamic workflows. Ask Claude to create a workflow and it orchestrates work across tens to hundreds of agents in the background, so you can take on larger, more complex tasks. Run /workflows to view your runs. (Trigger keyword is now ultracode as of v2.1.160.)
  • feature The lean system prompt is now the default for all models except Haiku, Sonnet, and Opus 4.7 and earlier.
  • feature claude agents: type ! <command> to run a shell command as a background session. Also claude --bg --exec '<command>'.
  • feature /simplify now runs a cleanup-only review (reuse, simplification, efficiency, altitude) and applies the fixes.
  • feature Claude in Chrome: pick which connected browser to use via /chrome → "Select browser…".
  • feature Streaming tool execution is now always enabled, including on Bedrock/Vertex/Foundry.
  • security Improved the auto-mode classifier's detection of data exfiltration, particularly bulk transfers of repository contents.

Claude Desktop

macOS · Windows

What's new

Sources: support.claude.com release notes, docs.claude.com platform overview, Anthropic news. No new Desktop-specific release in the last 7 days; items below remain current.

Customize panel. A unified Customize section in Claude Desktop groups Skills, Plugins, and Connectors in one place — the same screen used to install our engineering plugin and GitHub connector.
Browser handoff & console reads. Start a task in Claude Desktop and let it drive your browser via Claude in Chrome without switching windows. Claude can now also read browser console output, helping debug front-end errors without context switching. Workflow recording lets you teach Claude a multi-step browser sequence once and replay it.
  • feature Plugin marketplace and admin controls available on Team and Enterprise plans — pin or block plugins org-wide.
  • feature Single Customize directory for skills, connectors, and plugins (Settings → Customize).
  • feature Browser control from Claude Desktop: hand off web tasks to Claude in Chrome without context switching.
  • feature Workflow recording in Claude in Chrome: capture a sequence of browser steps and replay them as a skill.
  • feature Claude for Excel and Claude for PowerPoint add-ins now share conversation context bidirectionally.
  • feature Persistent agent thread — Pro and Max plans can manage Cowork tasks from a chat thread in Desktop.

Claude Cowork

Research preview

What's new

Sources: Get started with Claude Cowork, unified directory docs, Financial Services plugins, Use plugins in Claude. No new Cowork release in the last 7 days; items below remain current.

Scheduled tasks. Delegate work to Cowork on a recurring or on-demand basis — type /schedule in chat to create one. Available on all paid plans. This morning's digest is exactly this in action.
Projects. Organise related Cowork tasks into persistent, self-contained workspaces with their own files, links, instructions, and memory. Better for recurring or long-running work like this morning's daily digest.
  • feature Plugins bundle skills, connectors, and sub-agents into one installable package. Knowledge Work marketplace is default; Financial Services and Legal are available as add-ons.
  • feature Ten ready-to-run Financial Services agent templates (pitchbooks, KYC screening, month-end close) ship as plugins for Cowork and Claude Code.
  • feature Install custom plugin marketplaces from a GitHub repo.
  • feature Skills, Connectors, and Plugins all browseable from a unified directory in Settings.
  • feature Computer use — Claude can click, type, open apps, and run dev tools in Cowork with no setup. Research preview on Pro and Max plans.
  • feature Generally available on macOS and Windows via Claude Desktop.
  • feature Extend Cowork with third-party MCP platforms (Slack, Sentry, AWS, Atlassian, etc.).
