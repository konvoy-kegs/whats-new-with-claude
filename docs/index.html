Claude Fable 5

model · access restored

Access restored after export controls lifted headline

Source: Redeploying Claude Fable 5 · anthropic.com · posted 30 Jun, restored 1 Jul 2026

Fable 5 and Mythos 5 are back. On 30 Jun the US government lifted the export controls that had suspended both models since 12 Jun. Fable 5 returned on 1 Jul to users globally on the Claude Platform, Claude.ai, Claude Code, and Claude Cowork. Anthropic is re-enabling access on AWS, Google Cloud, and Microsoft Foundry “as quickly as possible” — those cloud paths may not be live yet. Mythos 5 has been restored for a set of US organisations (government approval on 26 Jun), with Glasswing access expanding from there. Access to all other models — Opus 4.8, Sonnet 5, Haiku — was never affected.
What this means for us: workflows pinned to claude-fable-5 on Claude Code and the API should work again. But Bedrock/AWS access is still being re-enabled — keep CI agents and Bedrock calls on claude-opus-4-8 until Fable 5 is confirmed live on AWS. Also note the new safety classifier (below) blocks more borderline requests during routine coding/debugging; any blocked request to Fable 5 is auto-routed to Opus 4.8, so callers should expect an occasional silent model switch on security-adjacent prompts.
  • status Export controls lifted 30 Jun; Fable 5 available again globally from 1 Jul across Claude Platform, Claude.ai, Claude Code, and Cowork. Cloud provider (AWS / Google Cloud / Microsoft Foundry) re-enablement is in progress, not confirmed complete.
  • safeguards Anthropic shipped an improved cybersecurity classifier that blocks the specific Amazon-reported bypass in >99% of cases; blocked requests are flagged and re-routed to Opus 4.8. It flags benign requests more often during routine coding/debugging — expect more false positives. US Commerce CAISI tested both the prior and new safeguards and rated them extraordinarily strong.
  • access & pricing For Pro, Max, Team, and select (premium) Enterprise plans, Fable 5 is included for up to 50% of weekly usage limits through 7 Jul, then via usage credits. Standard Enterprise seats have no included allowance — access is credits-only.
  • industry Anthropic, with Amazon, Microsoft, Google and other Glasswing partners, is drafting a consensus framework for scoring AI jailbreak severity, and has opened a HackerOne program for cyber-jailbreak submissions against Fable 5.
  • background Fable 5 launched 9 Jun as a Mythos-class model made safe for general use; Mythos 5 is the same model with fewer safeguards, restricted to Project Glasswing partners for defensive cybersecurity.

v2.1.198 — Claude in Chrome goes GA, plus a Claude Platform on AWS gateway provider newest

Published 1 Jul 2026 at 16:50 UTC · GitHub release · a top-level workflow going GA, an AWS gateway addition, a GitHub-native agent change, and a large reliability sweep

Claude in Chrome is now generally available. The browser agent that reads pages, drives web UIs, and hands work back to your terminal session moves out of beta — useful for our web-console and dashboard chores that don’t have a clean CLI.
Directly relevant to our AWS + GitHub stack. The gateway adds Claude Platform on AWS (anthropicAws) as an upstream provider, and model-not-found responses now advance the failover chain instead of dead-ending. Separately, background agents launched from claude agents now commit, push, and open a draft PR when they finish code work in a worktree — instead of stopping to ask — and awsAuthRefresh now runs automatically so Claude Platform on AWS / Mantle sessions no longer dead-end with “Please run /login” when the STS token expires.
  • workflow Claude in Chrome is now generally available.
  • gateway / AWS Added Claude Platform on AWS (anthropicAws) as an upstream gateway provider; model-not-found responses now advance the failover chain. awsAuthRefresh runs automatically so AWS / Mantle sessions no longer dead-end on STS token expiry.
  • background agents Background agents launched from claude agents now commit, push, and open a draft PR when they finish code work in a worktree; new background-agent notifications fire the Notification hook (agent_needs_input / agent_completed) when a session needs input or finishes.
  • feature New /dataviz skill for chart and dashboard design guidance with a runnable colour-palette validator; the built-in Explore agent now inherits the main session’s model (capped at opus) instead of running on haiku; subagents and context compaction now inherit the session’s extended-thinking config.
  • reliability Transient network drops mid-response (e.g. ECONNRESET) now retry with backoff instead of aborting the turn; background tasks in web, desktop, and VS Code panels no longer get stuck on “Running” after they finish; a teammate that dies on an API error now reports “failed” to the lead, and messaging a stuck teammate wakes it to retry; /diff now refreshes when you switch branches or commit outside the session.
  • fix Plan mode again auto-allows read-only tool calls when a session starts in plan mode; markdown tables no longer overflow their right border in fullscreen; /desktop no longer fails with “Cannot determine working directory” after exiting a worktree; the /agents wizard was removed — ask Claude to manage subagents, or edit .claude/agents/ directly.

v2.1.197 — Claude Sonnet 5 is now the default model in Claude Code

Published 30 Jun 2026 at 13:31 UTC · GitHub release · a major model release — update to v2.1.197 for access

Claude Sonnet 5 ships as the new default model. v2.1.197 makes Claude Sonnet 5 the default in Claude Code, with a native 1M-token context window and promotional pricing of $2 / M input and $10 / M output through 31 Aug 2026 (then $3 / $15). Anthropic positions it as its most agentic Sonnet yet — planning, tool use, and autonomous runs at a level that previously needed larger, pricier models, with performance close to Opus 4.8 at a fraction of the cost. Run claude update to pick it up. See Introducing Claude Sonnet 5 and the system card.
  • model Claude Sonnet 5 is now the default model in Claude Code, with a native 1M-token context window and promotional pricing of $2/$10 per Mtok through 31 Aug 2026. Update to v2.1.197 for access.
  • heads-up Because Sonnet 5 lands as the default, any session that hasn’t explicitly pinned a model in /model (or via an org default) will move off Opus 4.8 to Sonnet 5 on upgrade — pin a model first if you need a session to stay put. Pairs neatly with the org-default-model control added in v2.1.196.

v2.1.196 — organization default models & a committed settings file can no longer auto-spawn MCP servers

Published 29 Jun 2026 at 20:08 UTC · GitHub release · admin model governance, an MCP trust-boundary fix, and a background-reliability + watchdog sweep

Admins can now set an organization default model. Set it in the org console and it shows as “Org default” (or “Role default”) in /model for anyone who hasn’t explicitly picked a model themselves — a clean way to standardise the whole team on Opus 4.8, or pin a cheaper default, without touching each developer’s config. Pairs with the existing model-restriction / enforceAvailableModels controls.
claude mcp list/get no longer spawn MCP servers a repo self-approved via a committed .claude/settings.json. Previously a cloned repo could ship a committed .claude/settings.json that self-approved its own .mcp.json servers, and a plain claude mcp list/get would launch them. Untrusted workspaces now show ⏸ Pending approval until you explicitly approve — closing a quiet code-execution path when inspecting an unfamiliar repo. Directly relevant to anyone cloning shared or third-party repos.
  • trust boundary claude mcp list/get no longer spawn .mcp.json servers that a repo self-approved via a committed .claude/settings.json; untrusted workspaces show ⏸ Pending approval.
  • governance Organization default models — admins set it in the org console; it shows as “Org default” / “Role default” in /model when you haven’t picked one yourself.
  • feature Readable default session names at start (easier to identify and message); clickable file attachments in chat — Cmd/Ctrl-click reveals the file in Finder/Explorer.
  • behaviour change The streaming idle watchdog is now on by default for all providers — it aborts and retries when a response stream produces no events for 5 minutes (set CLAUDE_ENABLE_STREAM_WATCHDOG=0 to disable); Remote Control is now disabled when ANTHROPIC_BASE_URL points at a non-Anthropic host, matching the existing Bedrock/Vertex/Foundry behaviour.
  • reliability Background sessions now survive their process being stopped, restarted, or updated — including on Windows, where background shells are handed off instead of killed; agents killed by a daemon restart auto-resume; /code-review merged five cleanup finders into one (~25% fewer tokens); PowerShell git diff/git grep and quoted | patterns no longer reported as failures on exit 1; MCP OAuth invalid_scope fixed for GitLab self-hosted and other enterprise IdPs.

v2.1.195 — hook matchers now exact-match & project settings can’t auto-load external plugins

Published 26 Jun 2026 at 18:16 UTC · GitHub release · a hook-matching behaviour change, a plugin install-consent fix, and a background-agent reliability sweep

Hook matchers with hyphenated identifiers now exact-match — worth auditing our hooks. Previously a matcher like code-reviewer or mcp__brave-search could accidentally substring-match other identifiers; v2.1.195 makes them exact-match. If you want to match every tool from a hyphenated MCP server, use the explicit pattern mcp__brave-search__.*. Anyone running hooks keyed to hyphenated agent or MCP-server names should check they still fire as intended after upgrading.
A project’s .claude/settings.json can no longer silently enable an external plugin. Fixed a path where external plugins enabled only by a project’s .claude/settings.json didn’t require explicit install consent on every loader path — so cloning a repo that ships a crafted settings file can’t auto-load an external plugin without you approving it. Directly relevant to anyone cloning or running shared repos in Claude Code.
  • consent External plugins enabled only by a project .claude/settings.json now require explicit install consent on every loader path; /plugin Enable/Disable now works when a plugin’s plugin.json name differs from its marketplace entry name.
  • hooks Hook matchers with hyphenated identifiers (e.g. code-reviewer, mcp__brave-search) now exact-match instead of substring-matching — use mcp__brave-search__.* to match all tools from a hyphenated MCP server.
  • background agents Background jobs no longer disappear from claude agents or lose data when written by a newer Claude Code version; reopening a crashed background task no longer shows a blank screen for up to 5s; background-agent daemons no longer run unreachable when the control socket fails to start (which had blocked restarts).
  • feature New CLAUDE_CODE_DISABLE_MOUSE_CLICKS disables mouse click/drag/hover in fullscreen mode while keeping wheel scroll; Remote session startup now shows a provisioning checklist while the container starts.
  • voice Voice dictation on macOS no longer captures silence after the default input device changes; auto-submit now fires for languages written without spaces (Japanese, Chinese, Thai); Linux voice mode now distinguishes “no microphone” from “SoX not installed.”
