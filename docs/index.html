v2.1.163 — today's headline release

Published 4 Jun 2026 at 17:56 UTC · GitHub release

Pin the allowed Claude Code version org-wide. New requiredMinimumVersion and requiredMaximumVersion managed settings let admins define an allowed version range — Claude Code refuses to start if its version falls outside it and directs the user to an approved version. Useful for rolling out a known-good version across the team and blocking unvetted ones.
List installed plugins from the CLI. New /plugin list command shows installed plugins, with --enabled/--disabled filters — handy for auditing what's loaded across machines.
  • feature requiredMinimumVersion/requiredMaximumVersion managed settings — refuse to start outside the allowed version range and point to an approved version.
  • feature /plugin list command to list installed plugins, with --enabled/--disabled filters.
  • feature Hooks: Stop and SubagentStop hooks can now return hookSpecificOutput.additionalContext to give Claude feedback and keep the turn going without being labeled a hook error.
  • feature Added a "c to copy" shortcut to /btw that copies the raw markdown answer to the clipboard, preserving formatting when pasted elsewhere.
  • feature stdio MCP servers now receive the same CLAUDE_CODE_SESSION_ID as hooks/Bash on --resume.
  • feature Skills: added \$ escape syntax to include a literal $ before a digit in command bodies.
  • security Fixed org-managed permission rules not applying for the entire session when the managed-settings fetch completed during startup on a fresh config directory.
  • security Fixed deny rules on home-directory paths (e.g. Read(~/Desktop/**)) not blocking Bash commands that reference the path via $HOME.
  • fix claude -p no longer hangs forever after its final result when a backgrounded command never exits — background shells stop ~5s after the result once stdin closes.
  • fix claude -p no longer fails with "ANTHROPIC_API_KEY required" on Bedrock/Vertex/Foundry when CI=true and no Anthropic API key is set.
  • fix Bash commands no longer fail under bazel and EDR-protected Go workflows ($TMPDIR was wrongly overridden for all commands; regression in 2.1.154).
  • fix Windows: Bash commands no longer fail with "EEXIST" on the session-env directory when it's read-only or inside OneDrive.
  • fix Hook if: "Bash(...)" conditions no longer fire on every Bash command containing $() or $VAR; the pattern now matches commands inside subshells and backticks.
  • fix Background agent sessions now update to a new Claude Code version in the background, so opening a session after an update no longer waits on a cold restart.

v2.1.162 — previous user-facing release

Published 3 Jun 2026 at 18:09 UTC · GitHub release

See what background agents are blocked on. claude agents --json now includes a waitingFor field showing exactly what a waiting session is stuck on (e.g. a permission prompt). Handy when you have a dozen background sessions and need to find which one needs you.
Windows permission rules actually match now. Permission rules spelled with backslashes (~\, \\server\share) or case-variant paths previously never matched on Windows. Read deny rules also failed to hide files from Glob/Grep results. Both fixed.
  • feature claude agents --json exposes waitingFor to surface what a waiting session is blocked on.
  • feature Clicking a slash command in autocomplete now fills it into the prompt instead of running it; press Enter to run.
  • feature Renamed Windsurf to Devin Desktop in /ide, /terminal-setup, and /scroll-speed.
  • security Windows permission rules now match backslash paths and case-variant paths; Read deny rules now hide files from Glob/Grep results.
  • security WebFetch permission rules now apply to built-in preapproved domains — explicit WebFetch(domain:...) rules take precedence over the preapproved-host auto-allow.
  • fix Silent startup hang when the config directory is read-only/unwritable — now starts with in-memory config and surfaces errors.
  • fix MCP per-server timeout values below 1000 ms no longer get floored to a 1-second watchdog that aborted every tool call.

v2.1.161 — credential redaction in claude mcp

Published 2 Jun 2026 at 17:38 UTC · GitHub release

Credential leak fixed in claude mcp. claude mcp list/get/add no longer print secrets to the terminal: ${VAR} references are no longer expanded, and credential headers and URL secrets are redacted. Worth a glance if anyone has piped claude mcp output into logs, CI, or screenshots.
  • security claude mcp list/get/add no longer print secrets — ${VAR} references unexpanded; credential headers and URL secrets redacted.
  • feature OTEL_RESOURCE_ATTRIBUTES exported as labels on metric datapoints — slice usage by team or repo.
  • fix Parallel tool calls: a failed Bash command no longer cancels the other calls in the same batch.
  • fix Windows hooks that invoke bash explicitly (e.g. /usr/bin/bash script.sh) no longer fail with "command not found".

v2.1.154 — major release headline

Published 28 May 2026 at 15:40 UTC · GitHub release

Opus 4.8 is here. Defaults to high effort. Use /effort xhigh for your hardest tasks. Fast mode on Opus 4.8 is now 2× standard rate for 2.5× the speed.
Dynamic workflows. Ask Claude to create a workflow and it orchestrates work across tens to hundreds of agents in the background, so you can take on larger, more complex tasks. Run /workflows to view your runs. (Trigger keyword is now ultracode as of v2.1.160.)
  • feature The lean system prompt is now the default for all models except Haiku, Sonnet, and Opus 4.7 and earlier.
  • feature claude agents: type ! <command> to run a shell command as a background session. Also claude --bg --exec '<command>'.
  • security Improved the auto-mode classifier's detection of data exfiltration, particularly bulk transfers of repository contents.
