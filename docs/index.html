v2.1.162 — today's headline release

Published 3 Jun 2026 at 18:09 UTC · GitHub release

See what background agents are blocked on. claude agents --json now includes a waitingFor field showing exactly what a waiting session is stuck on (e.g. a permission prompt). Handy when you have a dozen background sessions and need to find which one needs you.
Windows permission rules actually match now. Permission rules spelled with backslashes (~\, \\server\share) or case-variant paths previously never matched on Windows. Read deny rules also failed to hide files from Glob/Grep results. Both fixed — worth checking if anyone on the team noticed surprising tool access on Windows boxes.
  • feature claude agents --json exposes waitingFor to surface what a waiting session is blocked on.
  • feature --tools: explicitly listing Grep/Glob now provides the dedicated search tools on native builds with embedded search.
  • feature /effort now confirms when your chosen level will persist as the default for new sessions.
  • feature Clicking a slash command in autocomplete now fills it into the prompt instead of running it; press Enter to run.
  • feature Remote Control now shows as a persistent footer pill (with session link) instead of a startup message.
  • feature Renamed Windsurf to Devin Desktop in /ide, /terminal-setup, and /scroll-speed, following the editor's rebrand.
  • security Windows permission rules now match backslash paths (~\, \\server\share) and case-variant paths; Read deny rules now hide files from Glob/Grep results.
  • security WebFetch permission rules now apply to built-in preapproved domains — explicit WebFetch(domain:...) deny/ask/allow rules take precedence over the preapproved-host auto-allow.
  • fix Silent startup hang when the config directory is read-only/unwritable — Claude Code now starts with in-memory config and surfaces errors instead of a blank screen.
  • fix Esc sent at the very start of a turn was being silently dropped in stream-json/SDK sessions, leaving the turn running with no "Interrupted" feedback.
  • fix API 400 no low surrogate in string errors when classifier side-queries or MCP server descriptions contain emoji near a truncation boundary.
  • fix MCP per-server timeout values below 1000 ms no longer get floored to a 1-second watchdog that aborted every tool call.
  • fix LSP workspaceSymbol now accepts a query parameter and returns results.
  • fix claude agents: live status text, session names, image paste, attach-after-restart, and lost-conversation/replies-on-failure issues all fixed.
  • fix Cross-session SendMessage no longer silently breaks when CLAUDE_CODE_TMPDIR or $TMPDIR is a deep path.
  • fix Background service startup and claude update verification now wait out endpoint-security scanning instead of failing after 5 seconds.
  • feature Quieter startup: notices group by severity, warnings rewritten to be shorter and clearer, failed turns now show a compact warning line instead of a multi-line red error block.

v2.1.161 — previous user-facing release

Published 2 Jun 2026 at 17:38 UTC · GitHub release

Credential leak fixed in claude mcp. claude mcp list/get/add no longer print secrets to the terminal: ${VAR} references are no longer expanded, and credential headers and URL secrets are now redacted. Worth a glance if anyone has piped claude mcp output into logs, CI, or screenshots.
Slice usage metrics by team or repo. OTEL_RESOURCE_ATTRIBUTES values are now attached as labels on metric datapoints, so you can break Claude Code usage down by custom dimensions like team or repository in your existing observability stack.
  • security claude mcp list/get/add no longer print secrets — ${VAR} references unexpanded; credential headers and URL secrets redacted.
  • feature OTEL_RESOURCE_ATTRIBUTES exported as labels on metric datapoints — slice usage by team or repo.
  • feature /mcp collapses claude.ai connectors you've never signed in to behind a "Show unused connectors" row.
  • feature claude agents rows show done/total when work is fanned out; peek surfaces the longest-running item.
  • fix Parallel tool calls: a failed Bash command no longer cancels the other calls in the same batch.
  • fix forceLoginOrgUUID/forceLoginMethod managed-settings policies no longer block third-party provider sessions (Bedrock, Vertex, Foundry, Mantle).
  • fix Workflow agents spawned with isolation: "worktree" in background sessions can now edit files inside their own worktree.
  • fix OpenTelemetry log events no longer dropped when emitted before telemetry init completes.
  • fix Windows hooks that invoke bash explicitly (e.g. /usr/bin/bash script.sh) no longer fail with "command not found".

v2.1.160 — write-path security prompts

Published 1 Jun 2026 at 20:03 UTC · GitHub release

New write-path security prompts. Claude now prompts before writing to shell startup files (.zshenv, .zlogin, .bash_login) and ~/.config/git/ — closing a quiet path to unintended command execution. acceptEdits mode also prompts before touching code-executing build-tool configs (.npmrc, .yarnrc*, bunfig.toml, .bazelrc, .pre-commit-config.yaml, .devcontainer/).
Workflow trigger renamed to ultracode. Typing the word "workflow" no longer triggers a dynamic-workflow run — the trigger keyword is now ultracode (highlighted in violet in the prompt input). Asking for a workflow in your own words still works.
  • security Prompt before writing to shell startup files and ~/.config/git/.
  • feature Edit no longer requires a separate Read after viewing a file with grep: single-file grep/egrep/fgrep now satisfy the read-before-edit check.
  • fix Restoring a completed claude agents session no longer drops chat history and re-runs the original prompt.
  • fix Copy-on-select now writes to the Windows clipboard on WSL.

v2.1.154 — major release headline

Published 28 May 2026 at 15:40 UTC · GitHub release

Opus 4.8 is here. Defaults to high effort. Use /effort xhigh for your hardest tasks. Fast mode on Opus 4.8 is now 2× standard rate for 2.5× the speed.
Dynamic workflows. Ask Claude to create a workflow and it orchestrates work across tens to hundreds of agents in the background, so you can take on larger, more complex tasks. Run /workflows to view your runs. (Trigger keyword is now ultracode as of v2.1.160.)
  • feature The lean system prompt is now the default for all models except Haiku, Sonnet, and Opus 4.7 and earlier.
  • feature claude agents: type ! <command> to run a shell command as a background session. Also claude --bg --exec '<command>'.
  • feature /simplify now runs a cleanup-only review (reuse, simplification, efficiency, altitude) and applies the fixes.
  • feature Claude in Chrome: pick which connected browser to use via /chrome → "Select browser…".
  • feature Streaming tool execution is now always enabled, including on Bedrock/Vertex/Foundry.
  • security Improved the auto-mode classifier's detection of data exfiltration, particularly bulk transfers of repository contents.
